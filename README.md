# Menu Service — Микросервис меню кофейни

**Menu Service** — это один из микросервисов в составе распределённой системы **Coffeeshop**. Отвечает за управление каталогом напитков и десертов: создание, чтение, обновление и удаление позиций меню.

> **Проект находится в активной разработке.** Архитектура и API могут меняться по мере добавления новых сервисов.

---

## Общая архитектура проекта Coffeeshop

Menu Service является частью экосистемы микросервисов:

- **API Gateway** (в планах) — единая точка входа для фронтенда (REST/HTTP)
- **menu-service** (этот сервис) — управление меню, база menu_db
- **order-service** (в разработке) — обработка заказов, база orders_db
- **user-service** (в планах) — авторизация и профили пользователей

**Межсервисное взаимодействие:** gRPC  
**Внешнее взаимодействие (через Gateway):** REST/HTTP + JSON

## Архитектура сервиса

Сервис построен по принципу **слоёной архитектуры** с чётким разделением ответственности:

1. **Handler** — принимает gRPC/HTTP запросы, конвертирует proto-модели во внутренние и обратно. Не содержит бизнес-логики.
2. **Service** — содержит бизнес-правила (например, "возвращать только доступные позиции", "цена не может быть отрицательной").
3. **Repository** — единственный слой, знающий о PostgreSQL. Выполняет SQL-запросы и маппит результаты в Go-структуры.
4. **Database** — инициализация пула подключений к БД.

Поток запроса: `Request → Handler → Service → Repository → PostgreSQL`

# Быстрый старт
## Требования
> Go 1.22+,

# Запуск
## Запустить базу данных:
> перейти в menu-service и выполнить команду:       
> docker-compose up -d
## Сгенерировать код из proto-файлов:
> task proto
## Установить зависимости и запустить сервис:
> go mod tidy       
>task run

### После запуска сервис слушает два порта:     
8081 - gRPC (для межсервисного общения)     
8080 - HTTP (демо-интерфейс для браузера)

## Использование
### Через браузер (демо)
Откройте http://localhost:8080 - вы увидите веб-интерфейс с текущим меню.   
Через gRPC-клиент
Используйте Postman, grpcurl или BloomRPC:
Адрес: localhost:8081
Proto-файл: proto/v1/menu.proto

### Доступные методы (gRPC)

| Метод | Описание | Тип |
|-------|----------|-----|
| `GetActiveMenu` | Получить список доступных позиций (stream) | Server Streaming |
| `GetItem` | Получить позицию по ID | Unary |
| `CreateItem` | Добавить новую позицию | Unary |
| `UpdateItem` | Обновить существующую позицию | Unary |
| `DeleteItem` | Удалить позицию | Unary |

##  Roadmap (Планы развития)

### Menu Service
-  Покрытие unit-тестами (Service, Repository с моками)
- Добавление категорий и фильтрации
- Пагинация для больших меню

### Экосистема Coffeeshop
- **order-service** — обработка заказов, управление остатками
- **user-service** — регистрация, авторизация, профили
- **API Gateway** — единая точка входа, проксирование REST → gRPC, CORS, rate limiting
- **Frontend** — вынос в отдельный репозиторий (React/Vue)
- **Nginx/Caddy** — reverse proxy для доступа без указания портов
- **CI/CD** — GitHub Actions для автоматического тестирования и деплоя
- **Observability** — централизованное логирование, трейсинг (OpenTelemetry), дашборды
